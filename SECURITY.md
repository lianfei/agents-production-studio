# Security Policy

## Supported Versions

Only the latest published version is actively supported for security fixes.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

When reporting a vulnerability, include:

- affected version
- impact summary
- reproduction steps
- suggested mitigation if available

Until a dedicated security contact is added, maintainers should treat any private repository contact channel as the preferred reporting path.

## Sensitive Data

Never commit:

- model API keys
- admin tokens
- internal corpora
- local logs or generated artifacts that may contain private data
